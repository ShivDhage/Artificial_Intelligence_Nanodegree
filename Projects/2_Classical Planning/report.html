Classical Planning Project Report

Overview

This report compares uninformed search, greedy heuristic search, and A* search on four air cargo planning problems. It evaluates how planning graph heuristics affect performance, scaling, and plan quality across increasing domain size.

Experimental Setup

Problems Evaluated

  • Problem 1: 2 airplanes, 2 cargo items, 2 airports → 20 domain actions
  • Problem 2: 3 airplanes, 3 cargo items, 3 airports → 72 domain actions
  • Problem 3: 2 airplanes, 4 cargo items, 4 airports → 88 domain actions
  • Problem 4: 2 airplanes, 5 cargo items, 4 airports → 104 domain actions

Search Algorithms Tested

Uninformed Search (No Heuristic):

  1. Breadth-First Search (BFS)
  2. Depth-First Graph Search (DFS)
  3. Uniform Cost Search (UCS)

Informed Search:

  1. Greedy Best-First Search with h_unmet_goals
  2. Greedy Best-First Search with h_pg_levelsum
  3. Greedy Best-First Search with h_pg_maxlevel
  4. Greedy Best-First Search with h_pg_setlevel
  5. A* Search with h_unmet_goals, h_pg_levelsum, h_pg_maxlevel, and h_pg_setlevel

Experiments include all problems, with planning graph A* heuristics evaluated where practical. Larger problems demonstrate where these heuristics become too expensive to finish in a reasonable time.

Key Findings

Scaling Trends: Greedy heuristics are highly scalable and remain practical across all tested problems. Uninformed search and A* with full planning graph heuristics become infeasible for larger domains.

Heuristic Cost vs. Benefit: Planning graph heuristics reduce node expansions but incur high evaluation cost. For Problems 3 and 4, h_pg_levelsum improves plan quality but often requires tens of seconds, while h_pg_maxlevel and h_pg_setlevel are impractical on the largest problem.

Plan Quality: Simple heuristics like h_unmet_goals provide fast solutions, but they can return suboptimal plans as domain complexity grows. Optimal planning requires A* or expensive planning graph heuristics.

Answers to Project Questions

Question 1: Restricted Domain (Few Actions, Real-Time Operation)

Recommended Algorithms:

  1. Greedy Best-First Search with h_unmet_goals
    • Fastest runtime on Problems 1–4
    • Low search overhead and low memory usage
    • Acceptable plan quality for small problems
  2. Breadth-First Search
    • Guaranteed optimal solution on small domains
    • Predictable search behavior for real-time evaluation

Rationale: With a small action set, the simple heuristic already performs well, and the cost of planning graph computation is not justified.

Question 2: Very Large Domains (e.g., UPS Routing)

Recommended Algorithms:

  1. Greedy Best-First Search with h_unmet_goals
    • Fastest growth in runtime
    • Minimal preprocessing and easy scaling
  2. Depth-First Search with pruning
    • Low memory usage
    • Use only when approximate solutions are acceptable
  3. Domain-specific or hierarchical planning
    • Better suited for very large industrial problems

Rationale: Planning graph A* heuristics become too expensive on larger problems, and uninformed search is not scalable. A simple heuristic provides the best practical tradeoff.

Question 3: Finding Optimal Plans

Recommended Algorithms:

  1. A* Search with admissible planning graph heuristics
    • h_pg_maxlevel and h_pg_setlevel are theoretically admissible
    • Guarantee optimality when they can be computed
  2. Uniform Cost Search
    • Reliable and optimal without heuristic design
  3. Greedy search with strong heuristic
    • Useful when fast approximate solutions are preferred

Rationale: Optimal planning requires search methods with admissible heuristics or cost-based expansion. Planning graph heuristics can provide better guidance, but at the cost of computation time.

Conclusion

This project shows that the best practical solution is often a simple heuristic search. Greedy Best-First Search with h_unmet_goals provides the best runtime across four problems, while planning graph heuristics are useful for reducing search but are often too expensive for larger domains.

  • Best overall: Greedy Best-First Search with h_unmet_goals
  • Best for optimality: A* with admissible planning graph heuristics on small problems
  • Most scalable: Greedy search with simple heuristics
